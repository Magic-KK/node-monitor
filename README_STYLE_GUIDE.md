# 📝 README 更新日志编写规范

## ⚠️ 重要规则（必须遵守！）

### 1️⃣ 版本顺序：**倒序排列**（最新在前）

**✅ 正确示例：**
```markdown
## 📝 更新日志

### v1.8.0 (2026-03-05) - 最新功能
- 最新的功能描述

### v1.7.0 (2026-03-05)
- 上一个功能

### v1.6.0 (2026-03-05)
- 再上一个功能
```

**❌ 错误示例：**
```markdown
### v1.0.0
### v1.1.0
### v1.2.0  ← 旧的在前面，错了！
```

---

### 2️⃣ 版本号必须同步更新

**需要更新的地方：**
1. **README 顶部徽章** - `![Version](https://img.shields.io/badge/version-vX.Y.Z-6C63FF.svg)`
2. **更新日志标题** - `### vX.Y.Z (日期) - 功能名称`
3. **package.json** - `"version": "X.Y.Z"`
4. **页面底部版本显示** - `SYSTEM VERSION X.Y.Z`

**✅ 正确示例：**
```markdown
![Version](https://img.shields.io/badge/version-v1.8.0-6C63FF.svg)

### v1.8.0 (2026-03-05) - CSV 导出报告功能
```

```json
// package.json
{
  "version": "1.8.0"
}
```

---

### 3️⃣ 更新日志格式模板

```markdown
### vX.Y.Z (YYYY-MM-DD) - 功能名称 Emoji

- ✨ **新增** 功能描述
- 🔧 **修复** bug 描述
- 🎨 **优化** 体验描述
- 📝 **文档** 更新描述
```

**常用 Emoji：**
- ✨ 新功能
- 🐛 Bug 修复
- 🎨 样式优化
- ⚡ 性能优化
- 📝 文档更新
- 🔧 配置修改
- ♻️ 代码重构
- 🚀 重要发布

---

### 4️⃣ 每次更新必须检查

**提交前检查清单：**
- [ ] README 顶部版本号已更新
- [ ] 更新日志按倒序排列（最新在前）
- [ ] 版本号与 package.json 一致
- [ ] 页面底部版本显示已更新
- [ ] 更新日志内容准确描述了新功能
- [ ] 没有重复的版本号
- [ ] 日期格式正确 (YYYY-MM-DD)

## ⚠️ 常见错误

### ❌ 错误 1：顺序混乱
```markdown
### v1.4.0
### v1.2.0  ← 跳过了 1.3.0
### v1.3.0  ← 顺序错了
```

### ❌ 错误 2：版本号不一致
- README 徽章：v1.4.0
- package.json：1.0.0
- 更新日志：v1.8.0
- 页面显示：v1.6.0

### ❌ 错误 3：重复版本
```markdown
### v1.2.0
...内容...

### v1.2.0  ← 重复了！
...内容...
```

## 🎯 牛开发注意事项

**每次自动任务完成后，必须：**

1. **更新 README 顶部徽章**
   ```markdown
   ![Version](https://img.shields.io/badge/version-vX.Y.Z-6C63FF.svg)
   ```

2. **在更新日志最前面添加新版本**（倒序！）
   ```markdown
   ### vX.Y.Z (日期) - 功能名称
   - 功能描述
   ```

3. **更新 package.json**
   ```json
   "version": "X.Y.Z"
   ```

4. **更新页面底部版本显示**
   ```javascript
   // app.js 或 index.html
   SYSTEM VERSION X.Y.Z
   ```

5. **提交前检查**
   - 所有版本号一致
   - 更新日志倒序排列
   - 没有重复版本

---

**记住：最新版本号永远在最前面！每次更新都要同步更新所有版本号！** 🚀
